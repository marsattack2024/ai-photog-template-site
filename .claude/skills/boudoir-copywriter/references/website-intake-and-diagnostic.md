# Website Intake And Diagnostic

Use this file before drafting homepage copy, landing pages, service pages, or long-form website rewrites.

## Goal
Do not jump straight into prose.
First gather context, identify missing conversion inputs, and preserve the strongest raw material from the current page.

## Step 1: Scrape / read / absorb what is already provided
Before asking questions, extract whatever the user already gave you:
- current page copy
- headlines and subheads
- offers / session-fee language
- inclusions
- CTAs
- testimonials / reviews
- location / service keyword
- brand phrases worth preserving
- claims, counts, or promises that require verification

Do not ask the user to repeat information they already pasted.

## Step 2: Run a five-part diagnostic
When reviewing current page copy, identify:

1. **Keep**
   - lines, themes, or phrases that are emotionally strong or brand-specific
2. **Cut**
   - generic, repetitive, weak, or studio-centered language
3. **Verify**
   - numbers, guarantees, policies, “all inclusive” claims, review counts, client counts, payment-plan claims
4. **Missing**
   - process clarity, proof, CTA clarity, privacy, keyword/location clarity, offer structure, guide introduction, FAQs
5. **Ask**
   - the next 3 to 5 questions with the highest leverage for a better draft

## Step 3: Ask focused questions only when needed
Prioritize these questions for boudoir website work:

### Core page questions
- What page is this exactly: homepage, service page, landing page, offer page?
- What is the main keyword and location you want the page to target?
- What is the primary CTA?

### Offer / structure questions
- What is the real booking flow after someone inquires?
- What is included in the session fee vs purchased later?
- Is this IPS / reveal-based, all-inclusive, or hybrid?

### Proof questions
- What reviews or testimonials can we use?
- Are there any numbers we can verify confidently?
- Are there specific proof themes you want emphasized: nervousness, wedding gift, milestone, body image healing, age diversity, privacy, etc.?

### Reassurance / policy questions
- What happens to the photos?
- Are images ever shared without consent?
- Are payment plans, privacy policies, reshoots, or satisfaction promises real and usable?

### Voice questions
- Should the page feel softer, balanced, or emotionally risky?
- Are there phrases already on the site that must stay?

## Step 4: Know when not to ask
Do not ask questions the source material already answers.
If the user pasted strong reviews, use them.
If the current copy already states location, offer, and CTA clearly, do not re-ask those.
If enough is present to write a strong first pass, state assumptions in one short note and proceed.

## Good question set example
When the user pasted most of the website already:
1. Is “all inclusive” literally true in the pricing, or is that session-only language?
2. Do you want the page to feel softer or keep more emotional edge?
3. Is there a verified number or review count we can use?
4. What happens to images / privacy after the session?

## Bad question set example
- Tell me about your brand.
- Who is your target audience?
- What do you offer?
These are too broad when the pasted material already answers them.

## Output after intake
Once enough context is gathered, either:
- provide a short audit + full rewrite, or
- provide section architecture first if the user wants structure before prose.
