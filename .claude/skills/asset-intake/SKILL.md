# Asset Intake

Use this before heavy page building when the quality and placement of images
will determine the site quality.

## Workflow

1. Inventory provided assets by folder, filename, dimensions, and format.
2. Group images by likely use:
   hero, about, gallery, service cards, testimonials, CTA backgrounds, social
   proof, detail/texture images.
3. Flag image risks:
   too small, blurry, awkward crop, heavy file, wrong orientation, duplicated
   pose, inconsistent editing style, privacy concern, or missing alt context.
4. Recommend exact placements in the site structure.
5. Produce a missing-asset list only for what blocks launch or weakens the first
   impression.

## Standards

- Hero images need enough negative space for text or a layout that keeps text
  separate.
- Gallery images should show variety: close, wide, detail, vertical, horizontal,
  emotional, polished final.
- Boudoir assets require extra privacy care and should avoid unintentional crops.
- Use `next/image` and prefer modern optimized formats when adding assets.

## Output

- Asset map by section
- Best hero candidates
- Images to avoid
- Missing assets
- Alt text direction
- Any compression or conversion recommendation
