# Exact output structure

Always present the deliverable in this order.

## 1. Input summary
- website
- genre
- offer
- objections to address
- benefits or differentiators to highlight
- exclusions or things not to mention
- assumptions, if any

## 2. Website insights
Brief bullets or a short paragraph covering:
- positioning and tone
- offer or CTA language
- process and experience clues
- products or deliverables
- privacy, styling, prep, or reassurance details
- any differentiators worth using

## 3. Welcome offer title slide
Format:
- headline
- optional subhead

Guidance:
- Usually make the headline offer-driven.
- Challenge-style formats are usually strong here, for example: `Claim [offer] - Can You Get 4/4 Right?`
- If the user wants a softer or more elegant style, keep the challenge mechanic but soften the wording.

## 4. Claim slide
Format:
- headline
- 2 to 4 short proof-oriented bullets

Guidance:
- Set up the logic of the quiz.
- Use this slide to normalize concerns, preview the reward, and reinforce trust.

## 5. Main quiz questions
Create exactly 4.

For each question use:
- Question X
- question text
- answer format if not standard multiple choice, for example: `select all that apply` or `true or false`
- answer options
- Statement:
- concise explanation, usually 70 to 100 words, starting with a bolded headline-style transition and tied to the right answer

Guidance:
- Main questions should usually feel answerable, not open-ended.
- Prefer myth, objection, trust, value, process, or timing questions.
- At least 3 of the 4 should usually have a clear best answer.
- Mix answer formats when helpful, such as including one select-all-that-apply question or one true-or-false question.
- An occasional `all of the above` structure is allowed when it creates a strong feature or benefit reveal.
- Avoid making the main 4 feel like an intake form or opinion poll.

## 6. Offer slide
Format:
- headline
- 3 short benefit bullets
- button text suggestion

Guidance:
- Reflect the actual offer.
- Keep the CTA specific.
- Reinforce the most motivating benefits discovered in the quiz.

## 7. Fair enough slide
Format:
- headline
- short reassuring paragraph
- second-chance CTA line

Guidance:
- Validate hesitation.
- Reframe the next step as low pressure and safe.
- Keep it empathetic, not needy.

## 8. Additional information collection
Format:
- name field
- phone field
- open text question: why do you want to do this photoshoot?

## 9. Optional example questions
Create 4 extra questions that fit the same studio.
Use the same question, options, and statement format, including answer-format labels when relevant.
Label them clearly as optional extras.

Guidance:
- These should still match the same challenge-style pattern.
- Use them to expand into adjacent objections, myths, or differentiators.

## 10. Short thank-you page copy
Format:
- headline
- 2 to 4 short lines covering gratitude and next steps

## 11. Short follow-up email
Format:
- subject line
- short body

## Formatting notes

- Use clean headings.
- Make it easy to copy into Typeform.
- Do not add long explanations outside the deliverable unless the user asked for rationale.
