# Intake

Use this when the project starts from a client brief, brand notes, image folder,
competitor examples, or a loose request like "make this site for a studio."

## Collect Or Infer

- Studio name and genre
- Location and service area
- Primary offer and secondary offers
- Audience and emotional buying context
- Brand palette and font direction
- Required pages
- Available images and missing images
- Reviews/testimonials
- Contact/CRM destination
- Supabase content needed
- SEO targets
- Launch deadline and domain

Ask at most two questions before drafting a plan. If context is missing, make
reasonable assumptions and label them.

## Build Plan Output

Return:

- Site direction in one paragraph
- Recommended page map
- Section rhythm for the homepage
- Section rhythm for core service pages
- Supabase tables needed
- Asset placement map
- Copy gaps
- Technical tasks
- Launch blockers

## Guardrails

- Keep the template generic for photography and creative-service studios.
- Do not turn the plan into agency, coaching, or software marketing.
- Do not add SaaS/admin features unless explicitly requested.
- Prefer a shippable studio site over a complex automation system.
